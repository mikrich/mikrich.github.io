Dawn of the Progressive Single Page App

Mark Dalgliesh

Senior ui developer at Seek

Progressive Enhancement

  • Hackathon Idea, now in development
  • Render Seek UI without the need for JavaScript on the front end

Motivation

  • Not aimed directly at users who have disabled JavaScript
  • All users are non JavaScript users while an application is loading
  • Deliver content without the need for an entire application to be loaded
  • suits large public facing applications
  • time to first tweet was a massive problem at twitter 2meg SPA

Approach

  • React (React Router, Server Side Rendering)
  • Resolves route, then sends down the view as a template string
  • Query strings used to track app state prior to loading
  • checkboxes to enable micro interactions

Example Form

  • method / action will create the query string
  • submit handler will prevent default behaviour once the app is loaded
<form method='get'
      action='/results'
      onSubmit={this.onSubmit.bind(this)}>

What does HTTP/2 mean for Front End Engineers?

Mark Nottingham

Chair of the IETF HTTP Working Group and has been closely involved in the development of HTTP/2

HTTP/2?

  • a major revision of the Web's protocol

differences

  • is binary, instead of textual
  • is fully multiplexed, instead of ordered and blocking
  • uses a single connection per origin
  • uses header compression to reduce overhead
  • allows servers to proactively push responses into client caches

What does this mean

  • improved network utilisation
  • less work for browser to figure out when/where to send requests
  • HTTP APIs, headers, methods and status codes remain the same
  • 15% improvement with no optimisation

what does it mean on the front end

  • reduce request minimisation techniques
  • e.g. spriting / inlining / concatenation

supported browsers

  • IE11, Firefox 37+, Chrome 41+, Safari 9 and Opera 29+
